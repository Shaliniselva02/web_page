HTML & Web Technology Textbook

Chapter 1: Introduction to HTML

+

Topic: What is HTML?

Explanation: HTML (HyperText Markup Language) is the standard markup language for documents designed to be displayed in a web browser. It defines the structure of web content through elements like headings, paragraphs, and links.

Key Points:

  • HTML is not a programming language - it's a markup language
  • Uses tags like <html>, <head>, and <body>
  • Current version is HTML5 (released in 2014)
Note: All HTML documents must start with <!DOCTYPE html> declaration.

Chapter 2: Basic HTML Document Structure

+

Topic: Essential HTML Elements

Explanation: Every HTML document has a basic structure that includes these key elements:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document Title</title>
</head>
<body>
    <!-- Content goes here -->
</body>
</html>

Element Breakdown:

  • <!DOCTYPE html> - Document type declaration
  • <html> - Root element
  • <head> - Contains meta information
  • <body> - Contains visible content

Chapter 3: HTML Text Elements

+

Topic: Text Formatting in HTML

Explanation: HTML provides various elements for text structure and formatting:

<h1>Main Heading</h1>
<h2>Subheading</h2>
<p>This is a paragraph.</p>
<p>This text is <strong>bold</strong> and <em>italicized</em>.</p>
<ul>
    <li>Unordered list item</li>
</ul>
<ol>
    <li>Ordered list item</li>
</ol>

Common Text Elements:

  • Headings: <h1> to <h6>
  • Paragraph: <p>
  • Bold: <strong> or <b>
  • Italic: <em> or <i>
  • Lists: <ul>, <ol>, <li>

Chapter 4: HTML Links and Images

+

Topic: Connecting Content with Links and Images

Explanation: HTML allows embedding hyperlinks and images using these elements:

<!-- Link to another page -->
<a href="https://example.com">Visit Example</a>

<!-- Link to email -->
<a href="mailto:contact@example.com">Email Us</a>

<!-- Image -->
<img src="image.jpg" alt="Description" width="200">

Key Attributes:

  • href - Specifies the link destination
  • src - Specifies the image source
  • alt - Provides alternative text for images
  • target="_blank" - Opens link in new tab
Best Practice: Always include alt text for accessibility and SEO.

Chapter 5: HTML Tables

+

Topic: Organizing Data with Tables

Explanation: Tables are created using these elements:

<table border="1">
    <tr>
        <th>Header 1</th>
        <th>Header 2</th>
    </tr>
    <tr>
        <td>Row 1, Cell 1</td>
        <td>Row 1, Cell 2</td>
    </tr>
    <tr>
        <td>Row 2, Cell 1</td>
        <td>Row 2, Cell 2</td>
    </tr>
</table>

Table Elements:

  • <table> - Defines the table
  • <tr> - Table row
  • <th> - Table header cell
  • <td> - Table data cell
  • <caption> - Table caption

Chapter 6: HTML Forms

+

Topic: User Input with Forms

Explanation: HTML forms collect user input through various controls:

<form action="/submit" method="post">
    <label for="name">Name:</label>
    <input type="text" id="name" name="name" required>
    
    <label for="email">Email:</label>
    <input type="email" id="email" name="email" required>
    
    <label for="password">Password:</label>
    <input type="password" id="password" name="password">
    
    <input type="submit" value="Submit">
</form>

Form Elements:

  • <form> - Container for form elements
  • <input> - Various input types (text, email, password, etc.)
  • <label> - Description for form controls
  • <select> - Dropdown list
  • <textarea> - Multi-line text input
Tip: Always pair <label> with form controls using the for attribute.

Chapter 7: Semantic HTML

+

Topic: Meaningful Structure with Semantic Elements

Explanation: HTML5 introduced semantic elements that clearly describe their meaning:

<header>
    <h1>Website Title</h1>
    <nav>
        <a href="/">Home</a>
        <a href="/about">About</a>
    </nav>
</header>
<main>
    <article>
        <h2>Blog Post Title</h2>
        <p>Article content...</p>
    </article>
</main>
<footer>
    <p>Copyright 2023</p>
</footer>

Key Semantic Elements:

  • <header> - Introductory content
  • <nav> - Navigation links
  • <main> - Primary content
  • <article> - Self-contained composition
  • <section> - Thematic grouping
  • <footer> - Footer content
Why Semantic HTML? Improves accessibility, SEO, and code maintainability.

Chapter 8: HTML Multimedia

+

Topic: Embedding Audio and Video

Explanation: HTML5 provides native multimedia support:

<!-- Audio -->
<audio controls>
    <source src="audio.mp3" type="audio/mpeg">
    Your browser does not support audio.
</audio>

<!-- Video -->
<video width="400" controls>
    <source src="video.mp4" type="video/mp4">
    Your browser does not support video.
</video>

<!-- YouTube Embed -->
<iframe width="560" height="315" 
    src="https://www.youtube.com/embed/dQw4w9WgXcQ" 
    frameborder="0" allowfullscreen>
</iframe>

Multimedia Elements:

  • <audio> - Embeds sound content
  • <video> - Embeds video content
  • <source> - Specifies media sources
  • <iframe> - Embeds external content
Warning: Always include fallback content for browsers that don't support these elements.

Chapter 9: HTML5 APIs

+

Topic: Advanced HTML5 Features

Explanation: HTML5 introduced powerful APIs for enhanced functionality:

<!-- Geolocation API -->
<button onclick="getLocation()">Get Location</button>
<p id="demo"></p>
<script>
function getLocation() {
    if (navigator.geolocation) {
        navigator.geolocation.getCurrentPosition(showPosition);
    }
}
function showPosition(position) {
    document.getElementById("demo").innerHTML = 
        "Latitude: " + position.coords.latitude + 
        "<br>Longitude: " + position.coords.longitude;
}
</script>

<!-- Local Storage -->
<script>
localStorage.setItem("name", "John");
alert(localStorage.getItem("name"));
</script>

Key HTML5 APIs:

  • Geolocation - Gets user's physical location
  • Web Storage - localStorage and sessionStorage
  • Canvas - Drawing graphics with JavaScript
  • Drag and Drop - Native drag-and-drop interface
  • Web Workers - Background script execution

Chapter 10: HTML Best Practices

+

Topic: Writing Clean, Accessible HTML

Explanation: Professional HTML development requires following these practices:

<!-- Good HTML Example -->
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Proper Document Structure</title>
</head>
<body>
    <header>
        <h1>Main Heading</h1>
        <nav aria-label="Main navigation">
            <ul>
                <li><a href="#home">Home</a></li>
            </ul>
        </nav>
    </header>
    <main>
        <article>
            <h2>Article Title</h2>
            <p>Content with proper <strong>semantics</strong>.</p>
        </article>
    </main>
    <footer>
        <p>&copy; 2023 My Site</p>
    </footer>
</body>
</html>

Best Practices:

  • Use proper document structure with DOCTYPE
  • Always include lang attribute
  • Use semantic elements where appropriate
  • Provide alternative text for images
  • Ensure proper heading hierarchy
  • Validate your HTML (W3C Validator)
  • Consider accessibility (ARIA attributes)
Pro Tip: Use the W3C Markup Validation Service to check your HTML for errors.
